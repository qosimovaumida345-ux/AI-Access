◈ INSTALLATION GUIDE

Deploy ShadowForge in 60 seconds

One script. Zero config. AI-powered project generation starts immediately.

01

Prerequisites

Make sure Python 3.10+ is installed.

PowerShell 
python --version
# Should output: Python 3.10+

Don't have Python? Download here →

02

Clone Repository

PowerShell 
git clone https://github.com/YOUR_USERNAME/shadowforge-os.git
cd shadowforge-os
03

Install Dependencies

PowerShell 
pip install -r requirements.txt
04

Configure API Keys

PowerShell 
copy app\config\.env.example app\.env
notepad app\.env

Add at least one free API key. Groq is recommended.

05

Launch ShadowForge

PowerShell 
python app/core/main.py
◈ The GUI will open. Tell the agent what to build!

FAQ

Does ShadowForge send my data anywhere? ▾
No. ShadowForge is a local application. It only contacts the AI provider APIs you configure. Your prompts go to those APIs (Groq, OpenRouter, etc.) — never to us. The app has no telemetry, no tracking, and no backend server.
Can it access my system files? ▾
No. The sandbox mode (enabled by default) restricts all file operations to ~/shadowforge_workspace/ only. System directories, SSH keys, browser data, and credentials are all blocked.
Is an internet connection required? ▾
Yes — to call the AI provider APIs. The UI and local file operations work offline, but AI responses require API access.
What if all providers fail? ▾
ShadowForge has an automatic fallback chain. It tries each configured provider in priority order. If all fail, it reports a clear error and the agent enters IDLE state — it never silently crashes.
Does it work without a GPU? ▾
Yes. All AI inference runs on remote provider servers. ShadowForge only runs the UI and orchestration logic locally. A basic laptop is enough.
